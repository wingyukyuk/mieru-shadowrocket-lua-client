# mieru-shadowrocket-lua-client

> A pure-Lua [mieru](https://github.com/enfein/mieru) proxy protocol client for [Shadowrocket](https://apps.apple.com/app/shadowrocket/id932747118) (Proof of Concept)

**正體中文版請參閱 [README.zh_TW.md](README.zh_TW.md)**

## Introduction

This project is a proof-of-concept client implementing the [mieru](https://github.com/enfein/mieru) v3 proxy protocol in pure Lua, designed to run as a [Shadowrocket](https://apps.apple.com/app/shadowrocket/id932747118) Lua backend on iOS.

The script fully implements mieru v3 TCP mode, including:

- **XChaCha20-Poly1305** AEAD encryption/decryption
- **Poly1305** message authentication code
- **ChaCha20** stream cipher
- **SHA-256** hash function
- **HMAC-SHA256** and **PBKDF2-SHA256** key derivation
- SOCKS5 connect request framing
- Implicit nonce increment scheme
- Session management and segment protocol handling

## ⚠️ Important Notes

1. **If Shadowrocket adds native mieru support in the future, please switch back to the built-in implementation.** Native components will always be faster and more stable than a Lua script.
2. This script was developed with AI assistance and manually verified.
3. Currently **only TCP mode with protocol version v3** is supported.
4. Since Shadowrocket's Lua environment does not expose native cryptographic APIs, all encryption is performed in pure Lua, resulting in **very low throughput**. Expect approximately **0.5–1 Mbps** on iPhone devices.

## Usage

### Prerequisites

- [Shadowrocket](https://apps.apple.com/app/shadowrocket/id932747118) installed on iOS
- A working mieru server (v3 protocol, TCP mode)

### Setup

1. Transfer `mieru-backend-allinone.lua` to your iOS device (via iCloud, AirDrop, a web server, etc.).

2. Open Shadowrocket and add a new node. Select **Lua** as the type.

3. Fill in the following fields:

   | Field | Description |
   |-------|-------------|
   | **Address** | Your mieru server address |
   | **Port** | Your mieru server port |
   | **User** | Your mieru account username |
   | **Password** | Your mieru account password |
   | **File** | Select `mieru-backend-allinone.lua` |

4. Leave other options at their default values (they are not used by the script).

5. Save the node and connect.

### Advanced Settings (Optional)

The script accepts additional parameters via Shadowrocket's `settings` object:

| Parameter | Description | Default |
|-----------|-------------|---------|
| `protocol_version` | Protocol version (only `v3` is supported) | `v3` |
| `time_offset_sec` | Time offset in seconds to compensate client/server clock skew | `0` |
| `debug` | Enable debug logging (`true` / `false`) | `false` |

## Troubleshooting

If connections fail or stall, use the diagnostic variant to investigate:

1. Switch from `mieru-backend-allinone.lua` to `mieru-backend-diagnostic-allinone.lua` in your Shadowrocket node settings.
2. Keep `protocol_version` at `v3` (default).
3. Reproduce a failed request.
4. Export or review Shadowrocket logs — look for lines containing `[mieru-diag]`.

### Common Issues

**Handshake completes but the remote peer immediately closes the connection:**

This usually indicates a clock skew between your device and the server. Try setting `time_offset_sec` to `120` or `-120` and test again.

**Very slow performance:**

- Make sure you are using `mieru-backend-allinone.lua` (not the diagnostic variant).
- Set `debug` to `false` (or leave it unset).
- The diagnostic variant and debug logging add significant overhead on iOS.

## Development

### File Structure

| File | Description |
|------|-------------|
| `mieru-core.lua` | Protocol core — all cryptographic algorithms, key management, session/segment framing |
| `mieru-backend.lua` | Shadowrocket Lua backend adapter — implements `wa_lua_on_*` callbacks |
| `mieru-backend-allinone.lua` | Merged single file of core + backend (**recommended for end users**) |
| `mieru-backend-diagnostic.lua` | Diagnostic variant with verbose `[mieru-diag]` logging |
| `mieru-backend-diagnostic-allinone.lua` | Merged single file of core + diagnostic backend |
| `mieru-standalone-test.lua` | Standalone test client using LuaSocket |
| `mieru-backend-mock-test.lua` | Mock test exercising `wa_lua_on_*` callbacks against a live server |
| `build.lua` | Build script to regenerate the allinone files |

### Building

The `*-allinone.lua` files are generated by merging `mieru-core.lua` into the backend files. To regenerate them:

```bash
lua build.lua
```

This wraps `mieru-core.lua` in a closure and concatenates it with each backend file, producing:

- `mieru-backend-allinone.lua`
- `mieru-backend-diagnostic-allinone.lua`

### Local Testing

Both test scripts require [LuaSocket](https://luarocks.org/modules/lunarmodules/luasocket) and a running mieru server.

**Standalone test** — validates the core protocol directly:

```bash
lua mieru-standalone-test.lua <host> <port> <username> <password> [target_host] [target_port] [protocol_version] [time_offset_sec]
```

Example:

```bash
lua mieru-standalone-test.lua 127.0.0.1 10910 myuser mypassword example.com 80 v3 0
```

Expected output: contains `HTTP/1.1 ...` and `TEST_PASS`.

**Mock callback test** — validates the Shadowrocket `wa_lua_on_*` callback flow:

```bash
lua mieru-backend-mock-test.lua <host> <port> <username> <password> [script_path] [protocol_version] [time_offset_sec]
```

Example:

```bash
lua mieru-backend-mock-test.lua 127.0.0.1 10910 myuser mypassword mieru-backend.lua v3 0
```

Expected output: contains `HTTP/1.1 ...` and `[mock-test] TEST_PASS`.

## Technical Details

### Cryptographic Algorithms

All algorithms are implemented in pure Lua with no external C modules or FFI:

| Algorithm | Purpose |
|-----------|---------|
| XChaCha20-Poly1305 | AEAD encryption/decryption (data transport) |
| HChaCha20 | Extended-nonce subkey derivation |
| ChaCha20 | Stream cipher |
| Poly1305 | Message authentication code (MAC) |
| SHA-256 | Hash function |
| HMAC-SHA256 | PRF for key derivation |
| PBKDF2-SHA256 | Password-based key derivation |

### Bit Operation Compatibility

The script auto-detects available bit operation methods in the Lua runtime, trying in order:

1. `bit32` library (Lua 5.2)
2. `bit` library (LuaJIT)
3. Lua 5.3+ native bitwise operators (`&`, `|`, `~`)
4. Pure arithmetic fallback (works on any Lua version)

### Protocol Scope

**Supported:**

- TCP CONNECT
- mieru stream protocol framing
- PBKDF2-SHA256 key derivation (v3 parameters)
- XChaCha20-Poly1305 with 24-byte implicit nonce

**Not supported:**

- UDP associate
